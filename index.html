Практическое задание

Анализ компьютерной информации средствами операционной системы (анализ системных файлов, хронология событий)

Цель: Научиться отслеживать активность пользователей, анализировать паттерны использования системных ресурсов и создавать отчеты для мониторинга безопасности в Windows и Linux.

Подготовка к работе

Важно! Перед началом работы убедитесь, что у вас есть права администратора (Windows) или sudo (Linux).

Настройка Windows

Включите аудит безопасности (запустите PowerShell от имени администратора):

PowerShell 
# Включить аудит входов и выходов
auditpol /set /subcategory:"Logon" /success:enable /failure:enable
auditpol /set /subcategory:"Logoff" /success:enable

# Включить аудит доступа к файлам
auditpol /set /subcategory:"File System" /success:enable /failure:enable

# Включить аудит использования привилегий
auditpol /set /subcategory:"Sensitive Privilege Use" /success:enable /failure:enable

Создайте рабочие папки:

Command Prompt 
mkdir C:\temp\analysis
mkdir C:\temp\analysis\reports

Настройка Linux

Установите необходимые инструменты:

Bash 
# Ubuntu/Debian
sudo apt update
sudo apt install htop iotop lsof net-tools

# CentOS/RHEL
sudo yum install htop iotop lsof net-tools

Создайте рабочие папки:

Bash 
mkdir -p ~/analysis/reports
chmod 755 ~/analysis

Задание 1: Анализ активности пользователей

Что нужно сделать:

Найти все входы и выходы пользователей за последнюю неделю
Определить паттерны рабочего времени
Выявить подозрительную активность (ночные входы, неудачные попытки)
Создать отчет с графиками активности

Как выполнить:

Анализ входов пользователей (PowerShell):

PowerShell 
# Успешные входы пользователей
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4624} | 
Select-Object TimeCreated, @{Name='User';Expression={$_.Properties[5].Value}}, @{Name='LogonType';Expression={$_.Properties[8].Value}} | 
Format-Table -AutoSize
PowerShell 
# Неудачные попытки входа
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4625} | 
Select-Object TimeCreated, @{Name='User';Expression={$_.Properties[5].Value}} | 
Format-Table -AutoSize
PowerShell 
# Экспорт данных для анализа
Get-WinEvent -FilterHashtable @{LogName='Security'; ID=4624,4625} | 
Select-Object TimeCreated, Id, @{Name='User';Expression={$_.Properties[5].Value}} | 
Export-Csv -Path "C:\temp\analysis\user_activity.csv" -NoTypeInformation
Объяснение: Event ID 4624 = успешный вход, 4625 = неудачная попытка входа. Данные экспортируются в CSV для создания графиков в Excel.

Анализ входов пользователей:

Bash 
# Последние входы пользователей
last -n 50
Bash 
# Неудачные попытки входа
lastb -n 20
Bash 
# SSH подключения
grep "sshd" /var/log/auth.log | grep "Accepted\|Failed" | tail -20
Bash 
# Экспорт данных
last -n 100 > ~/analysis/user_logins.txt
grep "sshd" /var/log/auth.log | grep "Accepted\|Failed" > ~/analysis/ssh_activity.txt
Объяснение: Команда `last` показывает историю входов, `lastb` - неудачные попытки, grep ищет SSH активность в логах.

Задание 2: Анализ процессов и производительности

Что нужно сделать:

Найти процессы, потребляющие больше всего ресурсов
Определить процессы, запущенные пользователями
Выявить подозрительные процессы
Создать отчет о производительности системы

Как выполнить:

Анализ процессов (PowerShell):

PowerShell 
# Топ процессов по памяти
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10 Name, Id, WorkingSet, CPU
PowerShell 
# Процессы с высоким CPU
Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name, Id, CPU, WorkingSet
PowerShell 
# Экспорт данных о процессах
Get-Process | Select-Object Name, Id, WorkingSet, CPU, StartTime | 
Export-Csv -Path "C:\temp\analysis\processes.csv" -NoTypeInformation
Совет: Откройте Диспетчер задач (Ctrl+Shift+Esc) для визуального анализа процессов.

Анализ процессов:

Bash 
# Интерактивный мониторинг процессов
htop
Bash 
# Топ процессов по CPU
ps aux --sort=-%cpu | head -10
Bash 
# Топ процессов по памяти
ps aux --sort=-%mem | head -10
Bash 
# Экспорт данных о процессах
ps aux > ~/analysis/processes.txt
ps aux --sort=-%cpu | head -20 > ~/analysis/top_cpu_processes.txt
Совет: В htop используйте F6 для сортировки, F9 для завершения процессов.

Задание 3: Анализ сетевых соединений

Что нужно сделать:

Найти все активные сетевые соединения
Определить процессы, использующие сеть
Выявить подозрительные соединения
Создать карту сетевой активности

Как выполнить:

Анализ сетевых соединений (PowerShell):

PowerShell 
# Активные TCP соединения
Get-NetTCPConnection | Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, State, OwningProcess
PowerShell 
# Соединения с именами процессов
Get-NetTCPConnection | Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, State, 
@{Name="ProcessName";Expression={(Get-Process -Id $_.OwningProcess -ErrorAction SilentlyContinue).ProcessName}}
PowerShell 
# Экспорт сетевых данных
Get-NetTCPConnection | Select-Object LocalAddress, LocalPort, RemoteAddress, RemotePort, State | 
Export-Csv -Path "C:\temp\analysis\network_connections.csv" -NoTypeInformation

Анализ сетевых соединений:

Bash 
# Активные соединения с процессами
ss -tulpn
Bash 
# Альтернативный способ
netstat -tulpn
Bash 
# Процессы с сетевой активностью
lsof -i
Bash 
# Экспорт сетевых данных
ss -tulpn > ~/analysis/network_connections.txt
lsof -i > ~/analysis/network_processes.txt
Объяснение: ss - современная замена netstat, lsof показывает открытые файлы и сетевые соединения.

Задание 4: Системная инвентаризация

Что нужно сделать:

Собрать информацию о системе и оборудовании
Получить список установленного ПО
Проанализировать конфигурацию системы
Создать полный отчет о системе

Как выполнить:

Системная информация (PowerShell):

PowerShell 
# Общая информация о системе
Get-ComputerInfo | Select-Object WindowsProductName, WindowsVersion, TotalPhysicalMemory
PowerShell 
# Информация о процессоре
Get-WmiObject -Class Win32_Processor | Select-Object Name, NumberOfCores, MaxClockSpeed
PowerShell 
# Установленные программы
Get-WmiObject -Class Win32_Product | Select-Object Name, Version, InstallDate | Sort-Object Name
PowerShell 
# Экспорт системной информации
Get-ComputerInfo | Export-Csv -Path "C:\temp\analysis\system_info.csv" -NoTypeInformation

Системная информация:

Bash 
# Информация о системе
hostnamectl
Bash 
# Информация о процессоре
lscpu
Bash 
# Информация о памяти
free -h
Bash 
# Дисковое пространство
df -h
Bash 
# Создание системного отчета
echo "=== System Report ===" > ~/analysis/system_report.txt
echo "Date: $(date)" >> ~/analysis/system_report.txt
echo "" >> ~/analysis/system_report.txt
hostnamectl >> ~/analysis/system_report.txt
echo "" >> ~/analysis/system_report.txt
lscpu | head -10 >> ~/analysis/system_report.txt

Контрольный список для отчета:

Результаты каждого задания со скриншотами

Выводы и рекомендации
Приложения с экспортированными данными
Совет по визуализации: Используйте Excel или LibreOffice Calc для создания графиков из экспортированных CSV файлов. Создайте временные диаграммы активности пользователей и круговые диаграммы использования ресурсов.

Пошаговая работа с экспортированными данными в Excel

Цель: Создать наглядные графики и отчеты на основе собранных данных для анализа активности пользователей и производительности системы.
Шаг 1: Подготовка данных
Откройте Excel и создайте новую книгу с названием "Анализ_системы_[дата]"
Импортируйте CSV файлы: Данные → Получить данные → Из файла → Из текста/CSV
Создайте отдельные листы: "Активность_пользователей", "Процессы", "Сеть", "Система"
Шаг 2: Анализ активности пользователей
Импортируйте user_activity.csv на лист "Активность_пользователей"
Создайте столбец "Час": Используйте формулу =ЧАС(A2) для извлечения часа из времени
Создайте столбец "День недели": Используйте формулу =ТЕКСТ(A2;"дддд") для получения дня недели
Создайте сводную таблицу: Вставка → Сводная таблица → Строки: Час, Значения: Количество записей
Постройте график активности по часам: Выделите сводную таблицу → Вставка → Гистограмма
Шаг 3: Анализ процессов
Импортируйте processes.csv на лист "Процессы"
Отсортируйте по WorkingSet: Выделите данные → Данные → Сортировка → По убыванию
Создайте топ-10 процессов: Скопируйте первые 10 строк в новую область
Постройте круговую диаграмму: Выделите Name и WorkingSet → Вставка → Круговая диаграмма
Добавьте подписи данных: Щелкните диаграмму → + → Подписи данных → Процент
Шаг 4: Анализ сетевых соединений
Импортируйте network_connections.csv на лист "Сеть"
Создайте сводную таблицу по портам: Строки: LocalPort, Значения: Количество соединений
Постройте график топ-портов: Выберите топ-15 портов → Гистограмма
Создайте таблицу внешних IP: Отфильтруйте RemoteAddress, исключив локальные адреса (127.*, 192.168.*, 10.*)
Шаг 5: Создание дашборда
Создайте лист "Дашборд" для размещения всех графиков
Скопируйте графики: Выделите график → Ctrl+C → Перейдите на "Дашборд" → Ctrl+V
Расположите графики: 2x2 сетка - активность пользователей, топ процессы, сетевые порты, системная информация
Добавьте заголовки: Вставка → Текстовое поле → "Анализ системы от [дата]"
Шаг 6: Создание отчетных таблиц
Создайте таблицу "Подозрительная активность": Входы в нерабочее время (22:00-06:00)
Создайте таблицу "Ресурсоемкие процессы": Процессы с потреблением памяти >100MB
Создайте таблицу "Внешние соединения": Все соединения с внешними IP-адресами
Добавьте условное форматирование: Выделите критичные значения красным цветом
Шаг 7: Финальное оформление
Создайте лист "Выводы" с текстовым анализом результатов
Добавьте рекомендации: Список действий для улучшения безопасности и производительности
Сохраните файл: Файл → Сохранить как → "Анализ_системы_[дата].xlsx"
Экспортируйте дашборд в PDF: Файл → Экспорт → Создать PDF/XPS
Полезные формулы Excel для анализа:
  • =ЧАС(A2) - извлечение часа из даты/времени
  • =ТЕКСТ(A2;"дддд") - получение дня недели
  • =СЧЁТЕСЛИ(B:B;">100000000") - подсчет процессов с памятью >100MB
  • =ЕСЛИ(ИЛИ(ЧАС(A2)<6;ЧАС(A2)>22);"Подозрительно";"Норма") - выявление активности в нерабочее время
Важно для безопасности: Все собранные данные содержат конфиденциальную информацию. Убедитесь, что отчеты хранятся в безопасном месте и доступны только авторизованным лицам.
